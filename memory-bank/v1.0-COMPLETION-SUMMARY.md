## Major Bug Fixes Completed

### Theme Switching Bug (Fixed)
**Problem:** Theme changes not applied in real-time, required app restart
**Solution:**
- Changed @State to @EnvironmentObject for ThemeManager
- Propagated theme via environment throughout view hierarchy
- Added @Published to trigger view updates
**Result:** Themes switch instantly, no restart needed

### Arabic Localization Bug (Fixed)
**Problem:** Arabic text displaying as boxes/question marks
**Solution:**
- Fixed Localizable.strings file encoding (UTF-8 with BOM ‚Üí UTF-8 without BOM)
- Re-saved files with proper encoding
- Validated Arabic Unicode characters
**Result:** Arabic displays correctly with proper glyphs

### Notification Bug (Fixed)
**Problem:** Notifications working in background but not foreground
**Solution:**
- Created AppDelegate with UNUserNotificationCenterDelegate
- Implemented foreground notification presentation
- Used @UIApplicationDelegateAdaptor in SwiftUI app
- Added userNotificationCenter(_:willPresent:) delegate method
**Result:** Notifications show banners and play sounds in foreground

### Countdown Timer Bug (Fixed)
**Problem:** Prayer time countdown frozen, not updating until tab switch
**Solution:**
- Added @MainActor isolation to PrayerTimesViewModel
- Wrapped timer callback in Task { @MainActor in }
- Changed objectWillChange.send() to direct @Published mutation
- Fixed race condition in timer updates
**Result:** Countdown updates live every second

### Calculation Method Bug (Fixed)
**Problem:** Changing calculation method didn't update prayer times
**Solution:**
- Added Combine observer on calculation method in SettingsViewModel
- Posted notification when method changes
- PrayerTimesViewModel listens and recalculates
- Added NotificationCenter.default.post
**Result:** Prayer times recalculate immediately when method changes

### Madhab Selection Bug (Fixed)
**Problem:** Madhab (Shafi/Hanafi) selection not affecting Asr time
**Solution:**
- Passed madhab parameter to Adhan library
- Used correct madhab enum values
- Verified Adhan integration
**Result:** Asr time correctly adjusts based on madhab

### Cardinal Markers Rotation Bug (Fixed)
**Problem:** N/E/W/S letters rotating with compass, becoming unreadable
**Solution:**
- Applied double counter-rotation technique
- ZStack rotates by -heading (letters move with compass)
- Individual Text elements rotate by +heading (stay upright)
- Net effect: Letters positioned correctly but stay readable
**Result:** Cardinal markers stay upright while compass rotates

### Text Color Adaptation Bug (Fixed - October 16)
**Problem:** Black text unreadable on dark theme backgrounds
**Solution:**
- Created textPrimary and textSecondary theme properties
- Added environment keys for theme text colors
- Updated .applyTheme() modifier to propagate text colors
- Replaced 40+ hardcoded Color("TextPrimary") with themeTextPrimary
- Fixed across all 11 affected view files
**Result:** All text adaptive and readable on all 4 themes

## Lessons Learned

### What Went Well ‚úÖ
1. **MVVM Architecture:** Clear separation of concerns made development smooth
2. **SwiftUI:** Rapid iteration, less code than UIKit, modern features
3. **Early Testing:** Unit tests caught edge cases early
4. **Adhan Library:** Saved 2-3 weeks of astronomical calculation work
5. **Early Localization:** RTL support from start prevented major refactoring
6. **Performance Profiling:** Instruments caught CPU bug that would have killed battery life
7. **Modular Views:** Small, reusable components easy to test and maintain
8. **Theme System:** Environment-based theming elegant and maintainable

### What Could Be Improved üí°
1. **Physical Device Testing:** Should have tested compass on real iPhone earlier (simulator limitations discovered late)
2. **Git Discipline:** Some commits too large, should have been broken down
3. **Documentation:** Should have documented architecture decisions as made (not after)
4. **Test Coverage:** Should have started with TDD for calculators
5. **Analytics Plan:** Should have researched privacy-friendly analytics earlier
6. **Beta Testing:** Should have planned TestFlight beta earlier
7. **Marketing Prep:** Should have started App Store assets sooner

### Key Technical Decisions

#### Why iOS 17.0+ minimum?
- **Pros:** Modern APIs (LocationButton, @Observable), smaller support burden, better performance
- **Cons:** Smaller user base (but growing rapidly)
- **Decision:** Worth it for development speed and modern UX

#### Why One-Time Purchase vs Subscription?
- **Pros:** User-friendly, no subscription fatigue, aligns with Islamic values (transparency, simplicity)
- **Cons:** Lower lifetime value per user
- **Decision:** User satisfaction > revenue maximization

#### Why Offline-First Architecture?
- **Pros:** Works in mosques (often poor signal), works in airplane mode, privacy-friendly, fast
- **Cons:** Cannot sync across devices, no cloud features
- **Decision:** Essential for reliability during prayers

#### Why No Analytics/Tracking?
- **Pros:** Privacy-focused, aligns with Islamic values, no GDPR concerns, user trust
- **Cons:** Limited debugging info, no usage insights, harder to optimize
- **Decision:** User privacy > data-driven decisions

#### Why MVVM vs Other Patterns?
- **Pros:** Standard SwiftUI pattern, clean separation, testable, Combine integration
- **Cons:** Boilerplate for simple views
- **Decision:** Best practice for SwiftUI apps
